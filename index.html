Arcanum AI - Intelligent Query Processor

Built for HackRX 6.0 Hackathon

A high-performance, resilient, and scalable Q&A system crafted late into the night from Aluva, Kerala. This project leverages a sophisticated hybrid AI strategy to deliver accurate answers from a vast range of document types.

🚀 Key Features

Hybrid AI Strategy

Automatically selects the best engine for the job:

  • Mission Agent: A focused, single-loop agent for individual questions requiring deep reasoning
  • Smart Batch Processor: An efficient engine for handling multiple questions, using different models based on query complexity

Advanced Caching System

A two-level caching system that drastically reduces latency on repeated requests:

  • L1 In-Memory Cache: For instant access to the most recently used document indexes
  • L2 PostgreSQL Cache: For persistent storage of document indexes, making first-time processing cost a one-off event

Resilient LLM Integration

Features automatic failover from more powerful models (Gemini 2.5 flash) to faster models (Gemini 2.0 Flash) to ensure high availability.

Comprehensive Document Support

Processes a wide array of formats:

  • PDFs (.pdf)
  • Word Documents (.docx)
  • PowerPoint Presentations (.pptx)
  • Excel Spreadsheets (.xlsx)
  • Web Pages (.html)
  • Images with text via OCR (.png, .jpg)

Additional Features

  • Content-Aware Routing: The batch processor analyzes query content for sensitive keywords and routes to the appropriate model
  • Production-Ready: Easy deployment with Gunicorn and professional Docker deployment options

🛠️ Technology Stack

  • Backend Framework: FastAPI
  • Web Server: Gunicorn + Uvicorn
  • Language Model: Google Gemini (1.5 Pro & 1.5 Flash)
  • Embeddings: SentenceTransformers (all-MiniLM-L6-v2 / all-mpnet-base-v2)
  • Vector Search: Facebook AI Similarity Search (FAISS)
  • Database: PostgreSQL
  • Document Parsing: PyMuPDF, python-docx, python-pptx, openpyxl, BeautifulSoup4
  • OCR: Tesseract
  • Deployment: Docker

⚙️ System Architecture

The application follows a modular, decoupled architecture:

  • User Request → FastAPI Web Server (Gunicorn + Uvicorn)
  • Master Processor (Orchestrator) routes to:
  • Single Question → Mission Agent → Gemini API
  • Multiple Questions → Smart Batch Processor → Gemini API (Pro/Flash)
  • Two-Level Cache (In-Memory + PostgreSQL)
  • Cache Miss → Document Processor → Vector Engine (FAISS) → Cache Write

🚀 Performance Features

  • Intelligent Routing: Automatically selects the most appropriate processing engine
  • Multi-level Caching: Reduces processing time for repeated queries
  • Batch Processing: Efficient handling of multiple questions
  • Failover Mechanism: Automatic fallback to ensure high availability
  • Vector Search: Fast similarity search using FAISS

CineShield - Anti-Piracy Content Protection

Team Members:

  • Gokul P - Lead, AI & Backend
  • Devika P Sajith - Backend Engineer
  • Ashna Thajudheen - Front-End

🎯 Elevator Pitch

A proof-of-concept anti-piracy system that uses a two-tiered analysis workflow. It combines a fast, low-cost perceptual hash (the "Sieve") for broad scanning with a powerful, high-accuracy AI model (the "Microscope") for final verification, ensuring both scalability and efficiency in detecting modified video content.

🚀 Key Features

  • Two-Tiered Detection System: Fast perceptual hashing combined with deep learning verification
  • Automated Scanning: Telegram bot and web scraper for continuous monitoring
  • Cross-Platform Support: Windows, Android, Linux (Debian/Ubuntu), and Universal AppImage
  • Real-Time Detection: Instant results for video analysis
  • Rapid Response: Quick identification and removal of pirated content

🏗️ Architecture Overview

Two-Tiered Detection System

  • Tier 1 (Sieve): Fast perceptual hashing for initial broad scanning
  • Tier 2 (Microscope): AI-powered deep analysis for final verification

Service Distribution

  • AI Engine: Local server combining Tier 1 and Tier 2 (exposed via Ngrok)
  • Middleware: Flask server orchestrating workflow and managing results
  • Front-End: Firebase-hosted web app + Flutter cross-platform app

⚙️ Workflow

  1. User uploads video through the front-end application
  2. Middleware receives the request and forwards it to the AI Engine
  3. AI Engine performs Tier 1 (perceptual hash) analysis
  4. If Tier 1 detects potential matches, Tier 2 (deep learning) verification is triggered
  5. Results are returned through the middleware to the front-end

🛠️ Technology Stack

  • Backend: Python 3.8+, Flask
  • AI/ML: Perceptual Hashing, Deep Learning Models
  • Front-End: Flutter SDK (cross-platform)
  • Hosting: Firebase (web), Local server with Ngrok tunneling
  • Automation: Telegram Bot API, Web Scraping

📦 Available Platforms

  • Windows (x86): Desktop application
  • Android: Mobile APK
  • Linux (Debian/Ubuntu): .deb package
  • Linux (Universal): AppImage for all distributions

✨ Latest Release - v1.0.0

First stable public release featuring:

  • Initial cross-platform support
  • Two-tiered piracy detection system
  • User-friendly interface for video analysis
  • Real-time detection results

🧪 Testing Methodology

Manual testing performed with:

  • Positive Test: Modified copies of source videos for match confirmation
  • Negative Test: Unrelated videos to verify correct dismissal
  • Edge Cases: Various modifications (cropping, filters, watermarks)
